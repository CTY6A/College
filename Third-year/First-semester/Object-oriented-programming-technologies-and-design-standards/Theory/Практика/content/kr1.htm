Контрольная работа №1

 

Анализ требований  и  построение  диаграмм вариантов использования (Use Case), топологии среды (Deployment), состояния объектов (STATECHART), деятельности (ACTIVITY)

 

Цель работы:

·      научиться строить диаграммы Use Case, Deployment, Statechart,  Activity в объектно-ориентированной среде автоматизированного синтеза;

·      разработать диаграммы  Use Case, Deployment, Statechart,  Activity для проектируемой системы.

 

Задание:

        описать функциональные требования к системе,  представить сценарии поведения ее объектов, их состояния, действия с помощью вышеперечисленных диаграмм.

 

Порядок выполнения работы

 

1. Изучить теоретическую часть лабораторной работы.

2. Ответить на контрольные вопросы.

       3. Разработать диаграммы Use Case, Deployment, Statechart,  Activity для проектируемой прикладной системы.

        

В качестве инструментальных средств для выполнения всех работ предложены две CASE-среды: Rational Rose и Rational XDE.

 

Контрольные вопросы

1.     Для чего используется диаграмма Use Case?

2.     Есть два прецедента, один из которых возникает при определенных обстоятельствах при выполнении другого. Какую связь нужно использовать для отражения такого взаимодействия:

a)  Include; б) Dependency; в) Extend.

3.     Для чего используется диаграмма Deployment?

4.     Каково назначение объектов Processor, Device, Connection?

5.    Почему в большинстве случаев элементы на диаграмме Deployment соединены связью Association, не имеющей направления?

6.     Какие отличия между диаграммами Activity и Statechart и что лучше моделировать с помощью каждой из диаграмм?

7.     Для чего используется элемент Swimlanes?

8.     Какая ситуация вызовет ошибку при проверке диаграммы с помощью режима Validate:

а) из значка Initial State выходят два перехода Transition; б) из значка End State выходит один переход Transition; в) все ответы правильны.

9.     Как отразить на диаграмме то, что несколько процессов должны быть завершены до перехода к следующему элементу деятельности:

а) использовать значок Proxy State;  б) использовать значок Decision;

в) использовать значок Swimlanes; г) ни один из перечисленных.
