Лабораторная работа №1

 

  Анализ требований  и  построение  диаграмм вариантов использования (Use Case)

 

Цель работы:

·      научиться строить диаграммы Use Case в объектно-ориентированной среде автоматизированного синтеза;

·      разработать диаграмму Use Case для проектируемой системы.

 

Задание:

        описать функциональные требования к системе и представить сценарии поведения ее объектов с помощью диаграммы Use Case.

 

Порядок выполнения работы

 

1. Изучить теоретическую часть лабораторной работы.

2. Ответить на контрольные вопросы.

       3. Разработать диаграмму Use Case по предложенной тематике.

        

В качестве инструментальных средств для выполнения всех работ предложены две CASE-среды: Rational Rose и Rational XDE.

 

Контрольные вопросы

1.     Для чего используется диаграмма Use Case?

2.    Какие значки специфичны только для диаграммы Use Case?

3.     Какой вариант не позволит создать новый элемент Actor на диаграмме прецедентов:

а) находясь в окне диаграммы Use Case, выбрать из контекстного меню Add UML => Actor;

б) щелкнуть по значку Actor в Toolbox, а затем на   диаграмме;

в) находясь в окне Model Explorer, из контекстного меню диаграммы Use Case выбрать Add UML => Actor.

4. Есть прецедент, который инициализируется актером. Какой тип связи обычно используется для их соединения:

a) Association;  б) Direct Association; в) Dependency.

5. Есть два прецедента, один из которых возникает при определенных обстоятельствах при выполнении другого. Какую связь нужно использовать для отражения такого взаимодействия:

a) Include; б) Dependency; в) Extend.
